# Sprint 765 — Automated Rollback Governance

Rollback triggers:

- elevated error rate;
- excessive p95 latency;
- availability degradation;
- repeated health-check failures;
- paused rollout state.

Governance controls:

- current and previous release identity;
- change ticket;
- accountable operator;
- deterministic rollback authorization ID;
- execution evidence;
- incident reference;
- root-cause evidence.
